# AKShare 持仓估值与候选基金筛选设计

日期：2026-05-27

## 目标

在现有基金监控脚本基础上增加两类能力：

- 根据实时估值自动计算已持有基金的估算金额、当日估算盈亏、总盈亏和收益率。
- 引入 AKShare 获取开放式基金排行数据，筛选未持有的候选基金，输出可关注原因。

## 范围

第一阶段包含：

- 配置支持 `holding_shares`、`cost_nav`、`target_profit_pct`、`max_loss_pct`。
- 持仓信号支持 `止盈观察`、`加仓观察`、`控制回撤`、`继续持有观察`。
- 新增 `screener` 模块，根据 AKShare 的开放式基金排行数据筛选候选基金。
- 新增 CLI 命令 `screen`，发送或 dry-run 输出候选基金列表。
- README 增加 AKShare 安装和候选筛选说明。

第一阶段不包含：

- 自动下单。
- 保证收益的买卖建议。
- 复杂机器学习预测。
- 读取支付宝账户或自动同步真实份额。

## 数据来源

- 持仓实时估值：继续使用当前天天基金实时估值接口。
- 候选基金池：优先使用 AKShare `fund_open_fund_rank_em(symbol="全部")`。
- 单只基金历史走势：后续可用 AKShare `fund_open_fund_info_em` 扩展。

## 规则语义

所有输出都是规则化观察信号：

- `可关注买入`：候选基金历史阶段收益和当日状态满足筛选条件。
- `加仓观察`：已持有基金低于成本或当日回调，但仓位风险可控。
- `止盈观察`：已持有基金收益率达到配置目标，适合检查是否分批止盈。
- `控制回撤`：亏损超过配置阈值，提示减少盲目加仓。
- `谨慎追高`：当日涨幅较大，不鼓励追涨。

## 测试

- 持仓份额和成本净值能计算估算金额、总盈亏和当日估算盈亏。
- 达到目标收益率时输出 `止盈观察`。
- 低于成本且当日回调时输出加仓观察类信号。
- 候选基金筛选能排除短期涨幅过高、长期收益过差、代码已持有的基金。
- 缺少 AKShare 时给出明确安装提示。
